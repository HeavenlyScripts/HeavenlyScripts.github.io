Introduction

Welcome to the Heavenly Docs.

This is the official documentation for the Heavenly UI Library, a Roblox Lua UI framework based off of Orion, with Orion compatibility. It gives you a full set of UI elements for building script interfaces fast, without boilerplate.

The library handles everything: window creation, tabs, elements, notifications, keybind tracking, a radial menu, a topbar, config saving, and startup animations. Every element returns an object you can call :Set() on to update it at runtime.

Note: Make sure your executor supports needed Functions. The library also uses writefile, readfile, isfile, and makefolder for config saving. These are optional but required for that feature.

Introduction

Installation

Load the library with loadstring and game:HttpGet in your executor. Call it once at the top of your script and store the result.

lua 
local Heavenly = loadstring(game:HttpGet("https://raw.githubusercontent.com/HeavenlyScripts/HeavenlyScripts.github.io/refs/heads/main/Heavenly.lua"))()

Legacy

If you encounter issues with the current release; such as unexpected UI behaviour, broken elements, or incompatibilities introduced by a recent update, you can load the previous stable version of the library instead.

lua 
local Heavenly = loadstring(game:HttpGet("https://raw.githubusercontent.com/HeavenlyScripts/HeavenlyScripts.github.io/refs/heads/main/HeavenlyLegacy.lua"))()

When to use this: Only switch to the previous version if the current release is actively causing problems in your script. The previous version does not receive bug fixes or new features. Once the issue is resolved in the main branch, switch back to the standard loadstring.

Script order matters. Always follow this structure; getting it wrong will cause features like the topbar and radial menu to silently not appear.

lua 
local Heavenly = loadstring(game:HttpGet("https://raw.githubusercontent.com/HeavenlyScripts/HeavenlyScripts.github.io/refs/heads/main/Heavenly.lua"))()

-- 1. Set feature flags FIRST, before Window
Heavenly.ShowTopbar = true
Heavenly.ShowRadial = true
Heavenly.RadialHotkey = Enum.KeyCode.Z
Heavenly.RadialMode = "hold"
Heavenly.ShowKeybindList = true

-- 2. Create the window
local Window = Heavenly:Window({ Name = "My Script" })

-- 3. Create tabs and add elements
local Tab = Window:Tab({
    Name = "LocalPlayer",
    Icon = "rbxassetid://110213647176253"
})

Tab:Toggle({
    Name = "Im a Toggle",
    Default = false,
    Color = Color3.fromRGB(255, 255, 255),
    Callback = function(state)
        print("Toggle is :", state)
    end
})

-- 4. Call Init (optional — auto-runs after a frame anyway)
Heavenly:Init()

Important: Heavenly.ShowTopbar, Heavenly.ShowRadial, and related flags must be set before Heavenly:Window(), not after. The library auto-calls Heavenly:Init() via task.defer after Window() returns, but calling it explicitly at the end is still fine and has no side effects.

Introduction

Theming

The library ships with two built-in themes: Dark and Light. Pass the theme name when creating the window. Custom themes must be registered with Heavenly.Themes:Add() before calling Heavenly:Window().

Common mistake: There is no Heavenly:SetTheme() method. Theme switching lives on the window object as Window:SetTheme(name) and takes a theme name string, not a table.

Built-in Themes

lua 
local Window = Heavenly:Window({
    Name = "My Script",
    Theme = "Dark",   -- "Dark" or "Light"
})

Custom Themes

Register your theme before the window, then pass the name to Theme. Any keys you omit fall back to Dark theme defaults.

lua 
-- Must be called BEFORE Heavenly:Window()
Heavenly.Themes:Add("MyTheme", {
    Main      = Color3.fromRGB(15, 15, 20),
    Second    = Color3.fromRGB(22, 22, 30),
    Stroke    = Color3.fromRGB(50, 50, 65),
    Divider   = Color3.fromRGB(50, 50, 65),
    Text      = Color3.fromRGB(240, 240, 240),
    TextDark  = Color3.fromRGB(130, 130, 150),
})

local Window = Heavenly:Window({
    Name  = "My Script",
    Theme = "MyTheme",
})

Switch Theme at Runtime

lua 
Window:SetTheme("Light")

Note: Heavenly:SetTheme() does not exist. Always call it on the window object.

Introduction

Window

The window is the root container for your entire UI. Create it with Heavenly:Window() after setting any feature flags.

lua 
local Window = Heavenly:Window({
    Name = "My Script",
    Theme = "Dark",
    Startup = true,
    StartupAnim = "Blob",
    StartupText = "Loading...",
    StartupIcon = "rbxassetid://...",
    CloseCallback = function() print("UI closed") end,
    SaveConfig = true,
    ConfigFolder = "MyScript",
    Config = tostring(game.GameId),
    ShowDisplayName = true,
    ShowUsername = true,
})

Note: StartupText & StartupIcon only work with the Typewriter animation.

Elements

Button

A clickable row that fires a callback. The returned object has a :Set() method to change the label text at runtime.

lua 
local Button = Tab:Button({
    Name = "Teleport to Spawn",
    Callback = function()
        game.Players.LocalPlayer.Character:SetPrimaryPartCFrame(CFrame.new(0, 5, 0))
    end
})

Button:Set("New Label")
OptionTypeDefaultDescription
Namestring"Button"Label shown on the row
CallbackfunctionnilCalled when clicked. Runs in a task.spawn.
Elements

Toggle

A checkbox element with a boolean value. Fires a callback with the new value on change. Supports flags for config saving and an optional inline confirmation prompt.

lua 
local toggle = Tab:Toggle({
    Name     = "Infinite Jump",
    Default  = false,
    Flag     = "InfiniteJump",
    Save     = true,
    Color    = Color3.fromRGB(0, 170, 255),
    Confirm  = false,   -- set true to require confirmation before toggling
    Callback = function(value)
        print("Infinite Jump:", value)
    end
})

toggle:Set(true)
print(toggle.Value)

ColorColor3RGB(9,99,195)Checkbox accent color when enabled

Bind

A keybind element. Click the row to enter binding mode, then press any key. Supports modifier keys and hold mode.

lua 
local bind = Tab:Bind({
    Name     = "Toggle Test",
    Default  = Enum.KeyCode.E,
    Hold     = false,
    Flag     = "ToggleUIBind",
    Save     = true,
    Callback = function()
        print("Bind fired!")
    end
})

bind:Set(Enum.KeyCode.F)
print(bind.Value)

Hold Mode: When Hold = true, the callback fires with true on key down and false on release.

A labeled group that contains other elements. Sections automatically resize to fit their children.

Features

Notify

Sends a stacked notification from the bottom-right of the screen. Multiple notifications stack automatically.

lua 
Heavenly:Notify({
    Name          = "Config Loaded",
    Content       = "Your saved config has been applied.",
    Time          = 5,
    Image         = "rbxassetid://4384403532",
    DurationColor = Color3.fromRGB(0, 200, 120),
})

-- Shorthand
Heavenly:Notify("Simple message", 3)

Common mistake: The duration key is Time, not Duration. Passing Duration is silently ignored.

Toast

Shows a small floating pill notification centered at the top of the screen, just below the Topbar. It slides in, waits 2 seconds, then fades out and destroys itself automatically. Multiple toasts can be shown at the same time.

lua 
-- Text only
Window:Toast("Settings saved!")

-- With icon (appears left of the text)
Window:Toast("Teleported!", "rbxassetid://137352523558390")
ArgumentTypeDefaultDescription
textstring""Message displayed in the pill
iconstringnilOptional asset ID. Icon is placed to the left of the text inside the pill.

Note: Toast lives in its own ScreenGui (HeavenlyToasts) so it renders above the window and topbar. It is not affected by the window being minimized or closed.

Features

Topbar

An optional floating panel at the top-center of the screen showing icon buttons for every tab, plus a search bar.

Must be set before Heavenly:Window(): ShowTopbar is read when the window is created. Setting it after won't work.

lua 
Heavenly.ShowTopbar = true   -- BEFORE Window()
local Window = Heavenly:Window({ ... })
Heavenly:Init()

Features

Flags & Config

Flags let you reference any element's value globally via Heavenly.Flags["FlagName"]. When SaveConfig = true and Save = true on an element, its value is written to a JSON file and auto-loaded on the next run.

lua 
local Window = Heavenly:Window({
    Name         = "My Script",
    SaveConfig   = true,
    ConfigFolder = "MyScript",
    Config       = tostring(game.GameId),
})

Tab:Toggle({
    Name     = "Speed Hack",
    Flag     = "SpeedHack",
    Save     = true,
    Default  = false,
    Callback = function(v) end,
})

print(Heavenly.Flags["SpeedHack"].Value)
Heavenly:Init()

Common mistake: Do not pass Config = true. Config is the filename string; SaveConfig is the boolean toggle.

Features

Lock / Unlock

Window:Lock() overlays the entire UI with a semi-transparent dimmed blocker and a centered message. All clicks are blocked while locked. Window:Unlock() fades the overlay out and removes it.

Useful for showing a loading state while your script does async work — lock the UI, do the work, then unlock.

lua 
Window:Lock("Loading data...")

task.spawn(function()
    task.wait(5)   -- simulate async work
    Window:Unlock()
end)
MethodArgumentDefaultDescription
:Lock(message?)string"Loading..."Message shown in the center of the overlay.
:Unlock()——Fades out and removes the lock overlay.

Note: Never call :Lock() on the main thread before your task.wait() or the UI will be unresponsive until Unlock is called. Always run the timed work in a task.spawn().

Features

Flash

Briefly pulses the window border stroke to draw the user's attention. The border brightens to the accent color (or a custom color), then fades back. Rapid repeated calls are safely ignored — the animation must finish before another Flash can start.

lua 
-- Uses the current accent color
Window:Flash()

-- Uses a custom color for this flash
Window:Flash(Color3.fromRGB(255, 200, 0))
ArgumentTypeDefaultDescription
colorColor3current accent colorOptional. Override the flash color for this call only. Does not change the accent color permanently.

Note: Flash is non-blocking — it returns immediately and animates in the background via task.delay. Calling Window:Flash() repeatedly in a tight loop with no wait won't compound; the guard flag drops extra calls while one is already running.

Features

Toggle Confirm

When Confirm = true is passed to a toggle, clicking it does not immediately fire the callback. Instead, the toggle row expands downward to reveal an inline "Are you sure?" row with Confirm and Cancel buttons. Clicking Confirm fires the toggle; Cancel collapses the row with no state change.

Use this for toggles that trigger irreversible or high-impact actions where accidental clicks would be disruptive.

lua 
Tab:Toggle({
    Name     = "Delete All Data",
    Default  = false,
    Confirm  = true,   -- enables the confirmation prompt
    Color    = Color3.fromRGB(200, 60, 60),
    Callback = function(state)
        if state then
            -- only runs after the user hits Confirm
            clearAllData()
        end
    end
})
OptionTypeDefaultDescription
ConfirmbooleanfalseWhen true, clicking the toggle shows a confirmation row before the callback fires.

Behaviour: The toggle frame smoothly expands from 38px to 76px. The confirm row clips its children until expanded, so the buttons are completely invisible until the user clicks the toggle. Clicking Cancel collapses it back with no state change. The checkbox only updates after Confirm is pressed.

Features

Destroy

Completely removes all Heavenly GUIs and clears all internal state. Use this when you want to fully clean up the library — for example, before reloading a script or switching executors.

lua 
Heavenly:Destroy()

Removes: HeavenlyUI, HeavenlyNotifications, HeavenlyKeybindList, HeavenlyTopbar, HeavenlyRadial, HeavenlyToasts. Also clears all Binds, Tabs, ElementRegistry, and internal references.
