Introduction

Welcome to the Heavenly Docs.

This is the official documentation for the Heavenly UI Library, a Roblox Lua UI framework based off of Orion, with Orion compatibility. It gives you a full set of UI elements for building script interfaces fast, without boilerplate.

The library handles everything: window creation, tabs, elements, notifications, keybind tracking, a radial menu, a topbar, config saving, and startup animations. Every element returns an object you can call :Set() on to update it at runtime.

Note: Make sure your executor supports needed Functions. The library also uses writefile, readfile, isfile, and makefolder for config saving. These are optional but required for that feature.

Introduction

Installation

Load the library with loadstring and game:HttpGet in your executor. Call it once at the top of your script and store the result.

lua 
local Heavenly = loadstring(game:HttpGet("https://raw.githubusercontent.com/HeavenlyScripts/HeavenlyScripts.github.io/refs/heads/main/Heavenly.lua"))()

Script order matters. Always follow this structure; getting it wrong will cause features like the topbar and radial menu to silently not appear.

lua 
local Heavenly = loadstring(game:HttpGet("https://raw.githubusercontent.com/HeavenlyScripts/HeavenlyScripts.github.io/refs/heads/main/Heavenly.lua"))()

-- 1. Set feature flags FIRST, before Window
Heavenly.ShowTopbar = true
Heavenly.ShowRadial = true
Heavenly.RadialHotkey = Enum.KeyCode.Z
Heavenly.RadialMode = "hold" 
Heavenly.ShowKeybindList = true

-- 2. Create the window
local Window = Heavenly:Window({ Name = "My Script" })

-- 4. Create tabs and add elements
local Tab = Window:Tab({
    Name = "LocalPlayer",
    Icon = "rbxassetid://110213647176253"
 })

Tab:Toggle({
    Name = "Im a Toggle",
    Default = false,
    Color = Color3.fromRGB(255, 255, 255) -- Optional
    Callback = function(state)
        print("Toggle is :", state)
    end
})

-- 5. Call Init
Heavenly:Init()

Important: Heavenly.ShowTopbar, Heavenly.ShowRadial, and related flags must be set before Heavenly:Window(), not after. Setting them after the window won't work because Heavenly:Init() reads them only once.

Introduction

Theming

The library ships with two built-in themes: Dark and Light. Pass the theme name when creating the window. Custom themes must be registered with Heavenly.Themes:Add() before calling Heavenly:Window().

Common mistake: There is no Heavenly:SetTheme() method. Theme switching lives on the window object as Window:SetTheme(name) and takes a theme name string, not a table.

Built-in Themes

lua 
local Window = Heavenly:Window({
    Name = "My Script",
    Theme = "Dark",   -- "Dark" or "Light"
})

Custom Themes

Register your theme before the window, then pass the name to Theme. Any keys you omit fall back to Dark theme defaults.

lua 
-- Must be called BEFORE Heavenly:Window()
Heavenly.Themes:Add("MyTheme", {
    Main = Color3.fromRGB(15, 15, 20),
    Second = Color3.fromRGB(22, 22, 30),
    Stroke = Color3.fromRGB(50, 50, 65),
    Divider = Color3.fromRGB(50, 50, 65),
    Text = Color3.fromRGB(240, 240, 240),
    TextDark = Color3.fromRGB(130, 130, 150),
})

local Window = Heavenly:Window({
    Name = "My Script",
    Theme = "MyTheme",
})

Switch Theme at Runtime

Call :SetTheme() on the window object, not on Heavenly itself.

lua 
-- correct
Window:SetTheme("Light")

Note: Heavenly has no SetTheme method. Heavenly:SetTheme({ ... }) will error.

Introduction

Window

The window is the root container for your entire UI. Create it with Heavenly:Window() after setting any feature flags. It returns a window object used to create tabs and call runtime methods.

lua 
local Window = Heavenly:Window({
    Name = "My Script",
    Theme = "Dark",
    Startup = true,
    StartupAnim = "Blob",
    StartupText = "Loading...",
    StartupIcon = "rbxassetid://...",
    CloseCallback = function()
        print("UI closed")
    end,
    SaveConfig = true,
    ConfigFolder = "MyScript",
    Config = tostring(game.GameId),
    ShowDisplayName = true, 
    ShowUsername = true, 
})

Note:StartupText&StartupIcon will only work on TypewriterAnimation.

Tabs

Tabs are created from the window object. Each tab gets a sidebar button and its own scrollable content page. If you enable the topbar or radial menu, tabs also appear there automatically.

lua 
local Tab = Window:Tab({
    Name = "Main",
    Icon = "rbxassetid://...",
})

Dropdown

A collapsible list of options. Click to expand, click an option to select. Supports refreshing the option list and setting values at runtime.

lua 
local dropdown = Tab:Dropdown({
    Name = "Select Game Mode",
    Options = { "Normal", "Hard", "Insane" },
    Default = "Normal",
    Flag = "GameMode",
    Save = true,
    Callback = function(value)
        print("Selected:", value)
    end
})

dropdown:Set("Hard")
dropdown:Refresh({ "Easy", "Normal", "Hard" }, true)
print(dropdown.Value)

Bind

A keybind element. Click the row to enter binding mode, then press any key or mouse button. Supports modifier keys (Shift, Ctrl, Alt) and hold mode. Blacklisted keys like WASD and Escape cannot be bound.

lua 
local bind = Tab:Bind({
    Name = "Toggle Test",
    Default = Enum.KeyCode.E,
    Hold = false,
    Flag = "ToggleUIBind",
    Save = true,
    Callback = function()
        print("Bind fired!")
    end
})

bind:Set(Enum.KeyCode.F)
print(bind.Value)

Hold Mode: When Hold = true, the callback fires with true when the key is pressed down and false when released.

Elements

Section

A labeled group that contains other elements. Sections automatically resize to fit their children. All standard tab elements are available inside a section.

lua 
local section = Tab:Section("Movement")

section:Toggle({
    Name = "Infinite Jump",
    Default = false,
    Callback = function(v) print(v) end,
})

section:Slider({
    Name = "Walk Speed",
    Min = 16,
    Max = 250,
    Default = 16,
    Callback = function(v)
        game.Players.LocalPlayer.Character.Humanoid.WalkSpeed = v
    end
})

Sections support all the same elements as a tab directly: :Button(), :Toggle(), :Slider(), :Dropdown(), :TextBox(), :Bind(), :Colorpicker(), :Label(), :Paragraph().

Features

Notify

Sends a stacked notification from the bottom-right of the screen. Notifications slide in, drain a timer bar, then slide out. Multiple notifications stack automatically.

lua 
Heavenly:Notify({
    Name          = "Config Loaded",
    Content       = "Your saved config has been applied.",
    Time          = 5,
    Image         = "rbxassetid://4384403532",
    DurationColor = Color3.fromRGB(0, 200, 120),
})
lua 
-- shorthand: text and duration as positional args
Heavenly:Notify("Simple message", 3)

Common mistake: The duration key is Time, not Duration. Passing Duration will be silently ignored and the notification will use the 5 second default.

Topbar

An optional floating panel at the top-center of the screen showing icon buttons for every tab, plus a search bar that finds any element or flag by name. The topbar is draggable and supports tab rows that expand past 6 icons.

Must be set before Heavenly:Window(): Setting ShowTopbar after the window is created will cause it to not appear. Feature flags are read once when Heavenly:Init() runs, but tabs are registered during Heavenly:Window(), so the flag must be set first.

lua 
-- Set BEFORE Heavenly:Window()
Heavenly.ShowTopbar = true

local Window = Heavenly:Window({ ... })
-- tabs and elements...
Heavenly:Init()

The search panel opens on click of the magnifying glass icon. It searches all flags and element names and navigates directly to the containing tab on selection.

Features

Radial Menu

A circular menu that maps each tab to a card arranged around a center point. Supports hold and toggle modes. Shows tab icons on the cards if the tab has an icon set.

Must be set before Heavenly:Window(): Same rule as the topbar; ShowRadial and RadialHotkey must be assigned before the window is created, not after.

lua 
-- Set BEFORE Heavenly:Window()
Heavenly.ShowRadial   = true
Heavenly.RadialHotkey = Enum.KeyCode.Z
Heavenly.RadialMode   = "hold"   -- "hold" or "toggle"

local Window = Heavenly:Window({ ... })
-- tabs and elements...
Heavenly:Init()
PropertyTypeDescription
ShowRadialbooleanEnable the radial menu
RadialHotkeyKeyCodeKey that opens/closes the wheel. Required if ShowRadial is true.
RadialModestring"hold" — wheel opens while key held, selects on release. "toggle" — key toggles open/close, click a card to select.

Keybind List

A persistent overlay panel in the bottom-left corner showing all active binds and their current keys. Updates automatically when a bind is changed. Only visible when at least one bind exists.

Features

Flags & Config

Flags let you reference any element's value globally via Heavenly.Flags["FlagName"]. When SaveConfig = true on the window and Save = true on an element, its value is written to a JSON file and auto-loaded on the next run via Heavenly:Init().

lua 
local Window = Heavenly:Window({
    Name = "My Script",
    SaveConfig = true,        -- boolean: enable saving
    ConfigFolder = "MyScript",  -- folder name
    Config = tostring(game.GameId),  -- string filename, NOT true
})

Tab:Toggle({
    Name = "Speed Hack",
    Flag = "SpeedHack",    -- registers to Heavenly.Flags
    Save = true,            -- included in save file
    Default = false,
    Callback = function(v) end,
})

-- read any flag value anywhere in your script
print(Heavenly.Flags["SpeedHack"].Value)

Heavenly:Init()  -- loads saved config if file exists

Common mistake: Do not pass Config = true. The Config option is the filename string, while SaveConfig is the boolean that enables the feature. Mixing these up will silently break config saving.

On load, if the config file exists, values are applied to all matching flags and a notification fires confirming the load. Config files are stored as JSON at ConfigFolder/Config.json using executor file functions (writefile, readfile, isfile, makefolder).

Features

Destroy

Completely removes all Heavenly GUIs from the game and clears all internal state. Use this for clean reloads or to wipe the UI on demand.

lua 
Heavenly:Destroy()

This removes the following GUIs: HeavenlyUI, HeavenlyNotifications, HeavenlyKeybindList, HeavenlyTopbar, HeavenlyRadial. It also clears Heavenly.Binds, Heavenly._Tabs, Heavenly._ElementRegistry, and all internal references.
